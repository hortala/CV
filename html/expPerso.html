Supervision de température (en cours)

Contexte:
Le principe de ce projet est de faire des relevés de température chaque n minutes puis de faire remonter ces informations sur un matériel de supervision.

Fonctionnement:
Des capteurs de température seront positionnés à plusieurs endroits dans la maison, ils remonteront des données sur la base de données via le web service.
Sur l'écran de supervision les informations au sujet des températures de chaque zone seront présentes.
Afin d'accéder à leurs informations les utilisateurs doivent se connecter avec leur mot de passe et leur identifiant.

Compétences utilisées:

  • Base de donnée postgresql
  • Service web en python
  • Supervision html/css/java script
  • Programmation language C pour arduino
  • Si gros projet (serveur MQTT)
  • Déploiement héroku (WebService / Base de donnée)

Porte Electronique gestion RFID

Contexte:
Le principe de ce projet est de restreindre l'accessibilité d'une zone à un badge ou carte RFID.
Cela permet aussi de savoir quelle personne a eu accès à cette zone et à quel moment.

Fonctionnement:
Afin de pouvoir déverrouiller la porte, l'utilisateur a besoin de présenter sa carte ou badge RFID. Une fois présenté le code RFID est comparé avec celui des utilisateurs recensés.
Pour une question de coût les informations sont stockées dans un fichier présent sur la carte SD.
Si la personne identifiée est un administrateur, la boîte contenant la carte SD se déverrouille afin qu'il puisse la récupérer et ajouter ou supprimer de nouveaux utilisateurs.

Amélioration:
L'axe principal d'amélioration que j'envisage est de connecter le système, à partir de ce moment-là j'utiliserai une carte Raspberry afin de créer une base de données locale, avec une application l'utilisateur pourra changer les informations sans avoir un accès physique.

Matériel utilisés:

  • Carte arduino nano
  • Lecteur de badge RFID
  • Lecteur de carte SD
  • Afficheur LCD
  • Module RTC (horloge temps réel)
